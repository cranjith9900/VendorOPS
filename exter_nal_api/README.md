# VendorOPS

FastAPI SAP adapter with business partner, bank account, SAP reference-table validation, PostgreSQL support, Swagger docs, automated endpoint tests, and a Postman collection.

## Run Locally

Go to the API folder:

```powershell
cd exter_nal_api
```

Install dependencies:

```powershell
python -m pip install -r requirements.txt
```

Run with SQLite:

```powershell
python -m uvicorn main:app --host 127.0.0.1 --port 8000
```

Run with PostgreSQL:

```powershell
$env:DATABASE_URL="postgresql://postgres@localhost:55432/sap_adapter_test"
python seed_postgresql.py
python -m uvicorn main:app --host 127.0.0.1 --port 8000
```

Swagger UI:

```text
http://127.0.0.1:8000/docs
```

## Tests

Go to the API folder first:

```powershell
cd exter_nal_api
```

Run all SQLite-backed endpoint tests:

```powershell
python -m pytest
```

Run PostgreSQL endpoint tests:

```powershell
$env:POSTGRES_TEST_DATABASE_URL="postgresql://postgres@localhost:55432/sap_adapter_test"
python -m pytest tests/test_postgresql_endpoints.py
```

Run live smoke tests against a running server:

```powershell
$env:BASE_URL="http://127.0.0.1:8000"
python smoke_test_endpoints.py
```

## Postman

Import `postman_collection.json` into Postman and run the collection against `http://127.0.0.1:8000`.

## How To Test Each Endpoint

Start the API before testing:

```powershell
cd exter_nal_api
python -m uvicorn main:app --host 127.0.0.1 --port 8000
```

Base URL:

```text
http://127.0.0.1:8000
```

You can test these endpoints in Swagger at:

```text
http://127.0.0.1:8000/docs
```

For PostgreSQL reference-table endpoints, seed the table data first:

```powershell
$env:DATABASE_URL="postgresql://postgres@localhost:55432/sap_adapter_test"
python seed_postgresql.py
python -m uvicorn main:app --host 127.0.0.1 --port 8000
```

### 1. Root Health Check

Use this endpoint to confirm the API server is running.

Method:

```text
GET
```

URL:

```text
http://127.0.0.1:8000/
```

Expected response:

```json
{
  "status": "SAP adapter API is running"
}
```

### 2. OAuth Token

Use this endpoint to get the dummy access token.

Method:

```text
POST
```

URL:

```text
http://127.0.0.1:8000/oauth/token
```

Body:

```text
No body required
```

Expected response:

```json
{
  "access_token": "dummy-token",
  "token_type": "bearer"
}
```

### 3. Create Business Partner

Create a business partner before testing the get business partner and bank account endpoints.

Method:

```text
POST
```

URL:

```text
http://127.0.0.1:8000/business_partner/
```

Headers:

```text
Content-Type: application/json
```

Body:

```json
{
  "bp_id": "BP001",
  "name": "Acme PostgreSQL Ltd"
}
```

Expected response:

```json
{
  "bp_id": "BP001",
  "name": "Acme PostgreSQL Ltd",
  "id": 1
}
```

Note: `id` is generated by the database, so the number may be different.

### 4. Get Business Partner

Fetch a business partner by `bp_id`.

Method:

```text
GET
```

URL:

```text
http://127.0.0.1:8000/business_partner/BP001
```

Expected response:

```json
{
  "bp_id": "BP001",
  "name": "Acme PostgreSQL Ltd",
  "id": 1
}
```

If the business partner does not exist, the response is:

```json
null
```

### 5. Create Bank Account

Create a bank account for an existing business partner.

Method:

```text
POST
```

URL:

```text
http://127.0.0.1:8000/bank/
```

Headers:

```text
Content-Type: application/json
```

Body:

```json
{
  "bp_id": "BP001",
  "account_number": "9876543210",
  "bank_code": "BANK01"
}
```

Expected response:

```json
{
  "id": 1,
  "bp_id": "BP001",
  "account_number": "9876543210",
  "bank_code": "BANK01"
}
```

Note: `id` is generated by the database, so the number may be different.

### 6. Get Bank Accounts

Fetch all bank accounts for a business partner.

Method:

```text
GET
```

URL:

```text
http://127.0.0.1:8000/bank/BP001
```

Expected response:

```json
[
  {
    "id": 1,
    "bp_id": "BP001",
    "account_number": "9876543210",
    "bank_code": "BANK01"
  }
]
```

If there are no bank accounts for the business partner, the response is:

```json
[]
```

### 11. Validate Reference Data - Valid Case

This checks whether company code, payment term, tax code, and bank master all exist.

Method:

```text
POST
```

URL:

```text
http://127.0.0.1:8000/tables/validate
```

Headers:

```text
Content-Type: application/json
```

Body:

```json
{
  "bukrs": "1000",
  "zterm": "NET30",
  "mwskz": "A1",
  "bankl": "BANK01"
}
```

Expected response:

```json
{
  "status": "VALID"
}
```

## Recommended Manual Test Order

Run the requests in this order when testing manually:

1. `GET /`
2. `POST /oauth/token`
3. `POST /business_partner/`
4. `GET /business_partner/BP001`
5. `POST /bank/`
6. `GET /bank/BP001`
7. `GET /tables/T001/1000`
8. `GET /tables/T052/NET30`
9. `GET /tables/T007A/A1`
10. `GET /tables/BNKA/BANK01`
11. `POST /tables/validate` with valid data
12. `POST /tables/validate` with invalid data
